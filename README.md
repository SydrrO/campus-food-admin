# 校园餐饮商家后台

版本：`v1.3`

这是餐饮商家后台的升级源码仓库，当前重点维护后台管理页面、优惠券管理、订单筛选和用户数据展示。仓库只保存源码，不保存数据库、证书、SSH 私钥或服务器密码。

## v1.3 主要内容

- 后台主页面标题和左侧名称更新为 `v1.3`。
- 用户列表仍默认折叠 0 消费用户，但搜索时会自动展示全部匹配用户，避免查不到 0 消费用户。
- 用户积分按已支付订单实付金额乘以 `100` 发放，每次支付入账时同步增加。
- 用户消费与积分刷新逻辑统一校准，退款或手动退款后会按有效订单重新计算。
- GitHub 改为每累计 5 个小版本统一上传和创建 Release。

## v1.2 主要内容

- 后台主页面标题和左侧名称更新为 `v1.2`。
- 用户“更多”弹窗里的电话改为从默认地址电话提取；没有下过单的用户电话留白。
- 用户列表默认折叠累计消费为 `0` 的用户，可手动展开/收起。
- 用户列表按累计消费金额降序展示，优先看到高消费用户。

## v1.1 主要内容

- 后台主页面标题和左侧名称更新为 `v1.1`。
- 订单处理标题旁新增“配送详情”按钮。
- 配送详情按当前订单筛选条件统计各配送区域的订单数，例如 `男7`、`书5`。
- 配送详情打开时会分页读取当前筛选下的完整订单结果，避免只统计第一页。
- GitHub 曾按每次更新同步创建版本标签和 Release；从 `v1.3` 起改为每 5 个小版本统一上传。

## v1 主要内容

- 后台整体风格改为蒂芙尼蓝轻奢风格。
- 新增优惠券发放页面，可按用户 ID 查询用户、发放优惠券、删除未使用优惠券。
- 优惠券列表默认隐藏已使用券，可通过弹窗查看已使用券。
- 订单页默认筛选：
  - 00:00 到 11:59 默认显示当天午餐。
  - 12:00 到 23:59 默认显示下一天午餐。
  - 状态默认显示已确认订单。
- 用户数据页移动端瘦身，只保留头像、四位数用户 ID、昵称、累计消费和“更多”按钮。
- 用户 ID 统一显示为四位数，例如 `1` 显示为 `0001`。
- 会员用户头像显示金色边框。
- “更多”弹窗显示电话、积分、订单数、创建时间、会员开通时间。
- 后台主页面标题和左侧名称显示 `v1`，方便和 GitHub 版本核对。

## 验收清单

1. 打开后台主页，左侧应显示 `赛杜后台 v1.3`。
2. 进入用户数据页，表格只展示头像、用户 ID、昵称、累计消费、操作。
3. 用户 ID 应为四位数格式，例如 `0001`。
4. 会员头像应有金色边框。
5. 点击用户行尾“更多”，弹窗应显示默认地址电话、积分、订单数、创建时间；未下过单的用户电话留白，会员用户额外显示会员开通时间。
6. 进入优惠券页面，已使用券默认不在列表中展示。
7. 点击“查看已使用券”，应弹窗展示该用户已使用优惠券。
8. 进入订单页，不手动选择筛选条件时，应按当前时间自动套用默认日期、午餐、已确认。
9. 进入订单页，点击“配送详情”，弹窗应按当前筛选显示每个配送区域分别几单。
10. 用户列表应默认折叠 0 消费用户，剩余用户按累计消费金额降序排列。
11. 搜索用户时，匹配到的 0 消费用户也应展示出来。
12. 新支付订单应按实付金额乘以 `100` 增加用户积分，退款刷新后积分应回到有效订单总额对应值。

## 部署说明

当前这版同时更新前端静态文件和会员积分服务，部署时应先备份服务器已有后台文件，再覆盖以下文件：

- `admin/index.html`
- `admin/js/workbench.js`
- `backend/app/services/membership.py`
- `backend/app/api/v1/endpoints/admin_orders.py`

部署后需要重启后端服务，并执行一次用户消费/积分重算脚本，让历史订单积分和新规则对齐。

## 注意事项

- 不要把密钥、数据库、证书、服务器密码提交到仓库。
- 不要覆盖服务器上无关项目。
- 每次上线前先做服务器备份，出问题可以直接回滚静态文件。
